# AWS STS

- AWS STS（Security Token Service）によって、外部ユーザーに対してAWSリソースへの一時的なセキュリティ認証情報を付与できる
- 有効期限を数分〜数時間（デフォルト3600秒、最小900秒〜最大12時間）に設定
- 認証情報はユーザーとともに保存されず、動的に生成、提供される
- sts:AssumeRoleアクションで、IAMロールから一時認証情報（アクセスキーID、シークレットアクセスキー、セッショントークン）を取得
- SAML2.0、OpenID Connect、IdP認証に対応

## スイッチロール

- 現在のログインセッションを維持したまま、一時的に別のIAMロールに切り替えること
- ユースケース
  1. 権限の分離と最小権限の原則 - 日常的なタスクから、より高い権限に一時的に権限昇華
  2. クロスアカウントでスイッチロール
